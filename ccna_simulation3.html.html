M RSTP and PVST+

  1. Enable rapid-pvst and set SW1 as root primary for VLANs 10 and 20. Name the show command that proves RSTP is active and the root per VLAN.
  2. Make user ports edge and enable BPDU Guard globally. Give the global commands and a show command to confirm an edge state.
  3. The cross-link between SW2 and SW3 should block. Name the show command that prints role and state on that interface.
  4. Set per-VLAN root secondary on SW3 for VLAN 20. Provide the single command and the verification.
Show answers

spanning-tree mode rapid-pvst; spanning-tree vlan 10,20 root primary; verify with show spanning-tree vlan 10,20

spanning-tree portfast default; spanning-tree portfast bpduguard default; verify with show spanning-tree interface detail

show spanning-tree interface Gi x/y

spanning-tree vlan 20 root secondary; verify with show spanning-tree vlan 20

N VTP and VLAN Pruning

  1. Configure VTP domain CCNA and mode server on SW1, clients on SW2 and SW3. Set a secure password. Name the show command to verify revision and mode.
  2. Enable VTP pruning so unused VLANs are not forwarded over trunks. Give the single command and the show that confirms pruning is on.
  3. Limit trunk to VLANs 10 and 20 on all inter-switch links. Provide the command and the show command to view the allowed list.
  4. Recover from a higher VTP revision on a new switch joining. Name the safest approach to prevent wiping VLANs and the command sequence.
Show answers

vtp domain CCNA; vtp mode server or client; vtp password xyz; verify with show vtp status

vtp pruning; verify with show vtp status

switchport trunk allowed vlan 10,20; verify with show interfaces trunk

Change to transparent and domain reset before joining, or clear config and set correct domain and lower revision

O Static and Floating Static

  1. Create a static route on R1 to 10.0.2.0/24 via 172.16.1.2 and on R2 back to 10.0.1.0/24. Give the show command that confirms installation.
  2. Add a floating static on R1 to 10.0.2.0/24 via a backup next hop 172.16.2.2 with AD 10 higher than the primary. Write the syntax and the verification.
  3. Track the primary next hop with IP SLA and make the floating static take over automatically. Name the SLA, track and static commands in order.
Show answers

ip route 10.0.2.0 255.255.255.0 172.16.1.2; verify with show ip route

ip route 10.0.2.0 255.255.255.0 172.16.2.2 10; verify with show ip route and show running-config

ip sla 1; icmp-echo 172.16.1.2; ip sla schedule 1 life forever start-time now; track 1 ip sla 1 reachability; ip route 10.0.2.0 255.255.255.0 172.16.2.2 track 1

P Extended ACL Scenarios

  1. Block HTTP and HTTPS from 10.0.1.0/24 to 10.0.99.10 but allow SSH and ICMP. Place the ACL closest to the source. Provide the three statements and the interface attachment.
  2. Permit only DNS queries out from LAN A and deny all other UDP to the server. Provide the two lines and the interface direction.
  3. Log denied Telnet attempts from any to the server. Provide the ACL line with logging and the show command that shows hit counts.
Show answers

ip access-list extended WEB-BLOCK; deny tcp 10.0.1.0 0.0.0.255 host 10.0.99.10 eq 80; deny tcp 10.0.1.0 0.0.0.255 host 10.0.99.10 eq 443; permit tcp any host 10.0.99.10 eq 22; permit icmp any host 10.0.99.10; apply ip access-group WEB-BLOCK in on LAN A interface

permit udp 10.0.1.0 0.0.0.255 any eq 53; deny udp 10.0.1.0 0.0.0.255 host 10.0.99.10 any; direction outbound toward server or inbound on LAN A depending on topology

deny tcp any host 10.0.99.10 eq 23 log; verify with show access-lists

Q SNMP NetFlow Syslog

  1. Configure SNMPv2c RO community netops restricted to 10.0.50.0/24. Give the single line and the show command to confirm.
  2. Enable SNMP traps for link up down to 10.0.0.50. Provide the two commands and one show command.
  3. Enable NetFlow on ingress of G0/1 and export to 10.0.0.60 on port 2055. Name the exporter and records in classic style.
  4. Send syslog to 10.0.0.70 facility local5 at informational and above. Name the verify command.
Show answers

snmp-server community netops RO 10; access-list 10 permit 10.0.50.0 0.0.0.255; verify with show snmp

snmp-server host 10.0.0.50 version 2c netops; snmp-server enable traps; verify with show snmp

ip flow-export destination 10.0.0.60 2055; ip flow-export version 9; interface g0/1 ip flow ingress; verify with show ip cache flow

logging host 10.0.0.70; logging facility local5; logging trap informational; verify with show logging

R IP SLA and Tracking

  1. Create IP SLA to ping 8.8.8.8 every 10 seconds and track it. If it fails, set a floating static default to use backup. List the exact command order.
  2. Verify SLA success rate and last failure time. Name the two show commands.
  3. Limit ICMP-based tracking to consider failure only after three consecutive misses. Name the command that adjusts threshold.
Show answers

ip sla 10; icmp-echo 8.8.8.8 source-interface G0/1; frequency 10; ip sla schedule 10 life forever start-time now; track 10 ip sla 10 reachability; ip route 0.0.0.0 0.0.0.0 203.0.113.1 5 track 10

show ip sla statistics; show track 10

delay down 30 up 10 on the track object
