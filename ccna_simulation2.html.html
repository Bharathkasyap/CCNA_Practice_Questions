G IPv6 and OSPFv3

Inter router link 2001:db8:12:12::/64 between R1 and R2. Hosts live in VLAN 10 using 2001:db8:10:10::/64.

  1. Assign IPv6 on the inter link and enable OSPFv3 process 1 in Area 0. Show the command that proves neighbors are Full.
  2. Enable Router Advertisements on R1 VLAN 10 so the host gets a global address and a default route with SLAAC. Name one show command on the host side that confirms the default route.
  3. Relay DHCPv6 from VLAN 10 to a server at 2001:db8:0:10::10. Name the SVI relay command and the show command that confirms bindings.
  4. Adjacency stuck in ExStart. Prove MTU mismatch with two show commands and give one interface command that allows adjacency without changing physical MTU.
  5. Advertise a default into OSPFv3 from R1 and name the show command on R2 that proves ::/0 is installed.
Show answers

show ipv6 ospf neighbor

Enable RA on SVI. On host: ip -6 route or netsh interface ipv6 show route

ipv6 dhcp relay destination 2001:db8:0:10::10 on SVI. Verify with show ipv6 dhcp binding

show ipv6 ospf neighbor and show interfaces. Fix with ipv6 ospf mtu-ignore on the link

default-information originate under OSPFv3 AF. Verify with show ipv6 route ::/0

H HSRP

Two routers provide a default gateway for VLAN 10 10.0.1.0/24. Virtual IP 10.0.1.254.

  1. Configure HSRP group 1 on both. Make R1 priority 110 and preempt. Show the one command that proves roles.
  2. Track the WAN on R1 and decrement 20 if it goes down so R2 becomes Active. Name the tracking command and the verification command.
  3. Fix a mismatch where one side uses HSRP v1 and the other v2. Name the knob and the show that reveals it.
  4. Enable simple HSRP authentication. Provide the line added on both and how you verify.
Show answers

show standby brief

standby 1 track G0/1 decrement 20, verify with show standby

standby version 2 on both, show standby shows version

standby 1 authentication cisco or md5 key string, verify with show standby

I EtherChannel with LACP

Link S1 and S2 using two FastEthernet links in a single bundle. Carry VLANs 10 and 20 only.

  1. Create LACP channel group 1 on both sides, set Port channel to trunk and allow VLANs 10 and 20. Name one show command that confirms bundle and members.
  2. One side is access, the other is trunk, the port is suspended. Name the show that exposes the mismatch and the change to fix it without removing the bundle.
  3. Increase LACP system priority so S1 wins aggregation. Name the global command and the show that displays system priority.
Show answers

show etherchannel summary

show interfaces switchport or show etherchannel detail. Fix with switchport mode trunk and switchport trunk allowed vlan 10,20

lacp system-priority 1. Verify with show lacp neighbor or show etherchannel detail

J Layer 2 Security

  1. Enable port security on an access port to allow one MAC, learn it dynamically and make it sticky. Violation shuts the port. Name the show command that lists the sticky MAC and state.
  2. Enable DHCP snooping for VLANs 10 and 20. Trust only the uplink. Name the command that prints the binding table.
  3. Enable Dynamic ARP Inspection for VLANs 10 and 20. Trust only the uplink, rate limit ARP on access ports, and give the verification command that shows inspected and dropped frames.
  4. Enable IP Source Guard on user ports so only DHCP snooping bindings are permitted. Name the interface command and one show to verify.
Show answers

switchport port-security; switchport port-security maximum 1; switchport port-security mac-address sticky; switchport port-security violation shutdown; verify with show port-security interface

ip dhcp snooping and ip dhcp snooping vlan 10,20; trust on uplink; show ip dhcp snooping binding

ip arp inspection vlan 10,20; trust uplink; ip arp inspection limit rate; verify with show ip arp inspection statistics

ip verify source on interface; verify with show ip verify source

K Management

  1. Enable SSH v2 only. Create a local user and restrict VTY to SSH. Name two show commands that confirm SSH is active and the version.
  2. Configure NTP with a reliable server and prefer it. Give the commands that prove sync and reachability.
  3. Send syslog to 10.0.0.50 with facility local7 and severity warnings or higher. Name the show that proves messages are being sent.
Show answers

ip domain-name, crypto key generate rsa, ip ssh version 2, line vty transport input ssh, login local; verify with show ip ssh and show ssh

ntp server 198.51.100.1 prefer; verify with show ntp associations and show ntp status

logging host 10.0.0.50, logging trap warnings, logging facility local7; verify with show logging

L NAT Pool and Static

  1. Create a NAT pool using 203.0.113.32/29 and overload to that pool for inside users 10.0.1.0/24. Name one show command that confirms translations.
  2. Create a static one to one NAT from 10.0.1.100 to 203.0.113.34. Give the verification command.
  3. Users report intermittent failures. The pool is exhausted. Show the command that proves the counters and give one change to restore service without new addresses.
Show answers

ip nat pool PUBLIC 203.0.113.32 203.0.113.38 netmask 255.255.255.248; ACL for inside; ip nat inside source list 1 pool PUBLIC overload; verify with show ip nat translations

ip nat inside source static 10.0.1.100 203.0.113.34; verify with show ip nat translations

show ip nat statistics shows misses; fix by switching overload to interface, reducing timeouts, or expanding pool
