S Wireless Multi-SSID

  1. Create two SSIDs Corp and Guest mapped to VLANs 10 and 20. Trunk the AP uplink and permit only 10 and 20. Name the show command on the switch that proves tagging.
  2. On the AP, secure Corp with WPA2-PSK and isolate Guest clients. Name the config lines and the show that proves both SSIDs are broadcasting.
  3. Guest cannot reach the internet because the SVI for VLAN 20 has no default route. Provide a minimal L3 fix and the verification command.
Show answers

switchport mode trunk; switchport trunk allowed vlan 10,20; verify with show interfaces trunk

dot11 ssid Corp auth open key-management wpa2 wpa-psk; dot11 ssid Guest guest-mode ap isolation; verify with show dot11 associations or show running-config section ssid

ip route 0.0.0.0 0.0.0.0 next-hop or enable NAT gateway; verify with show ip route

T Switch Hardening

  1. Disable unused ports, set them to access mode in a black hole VLAN 999, and shut them. Provide the interface range commands and a single show that confirms.
  2. Enable storm-control on access ports to limit broadcast to 2 percent and multicast to 2 percent. Provide the interface-level commands and a show to verify.
  3. Enable errdisable recovery for bpduguard and link-flap after 300 seconds. Provide the global command and a show to confirm timers.
  4. Disable CDP on user-facing ports but keep LLDP. Provide the commands and the show that proves CDP is off.
Show answers

interface range, switchport mode access, switchport access vlan 999, shutdown; verify with show interfaces status

storm-control broadcast level 2; storm-control multicast level 2; verify with show storm-control

errdisable recovery cause bpduguard link-flap; errdisable recovery interval 300; verify with show errdisable recovery

no cdp enable on interfaces; lldp run; verify with show cdp neighbors and show lldp neighbors

U PPP and PPPoE Basics

  1. Configure PPPoE client with Dialer1 using CHAP user cpe1 and password Cisco123. Provide the essential dialer and interface commands and the show that proves it is up.
  2. Switch authentication to PAP. Provide only the lines that change and one show that confirms the method.
  3. Set a dialer idle-timeout of 300 seconds and verify the session drops after idle.
Show answers

interface Dialer1, encapsulation ppp, dialer pool 1, ppp chap hostname cpe1, ppp chap password 0 Cisco123; interface e0/0 pppoe enable group global, pppoe-client dial-pool-number 1; verify with show pppoe session and show interface Dialer1

ppp authentication pap; ppp pap sent-username cpe1 password 0 Cisco123; verify with show ppp multilink or debug ppp authentication

dialer idle-timeout 300; verify with show interface Dialer1 and timers

V Router-on-a-Stick with DHCP

  1. Build subinterfaces for VLANs 10 and 20 and serve DHCP for both using pools with default routers 10.0.10.1 and 10.0.20.1. Provide the minimal pool lines and one show that lists bindings.
  2. Block inter-VLAN traffic but allow both to reach the internet via NAT on G0/1. Provide one ACL attached to each SVI direction and the NAT rule.
  3. Users report wrong DHCP scope on VLAN 20. Provide the show that reveals the wrong pool and the fix line.
Show answers

interface g0/0.10 encapsulation dot1Q 10 ip address 10.0.10.1 255.255.255.0; interface g0/0.20 encapsulation dot1Q 20 ip address 10.0.20.1 255.255.255.0; ip dhcp pool V10 network 10.0.10.0 255.255.255.0 default-router 10.0.10.1; ip dhcp pool V20 network 10.0.20.0 255.255.255.0 default-router 10.0.20.1; verify with show ip dhcp binding

ACL on each SVI to deny ip any 10.0.0.0 0.255.255.255 except to outside; ip nat inside on g0/0 subifs, ip nat outside on g0/1, ip nat inside source list 100 interface g0/1 overload

show running-config | section dhcp or show ip dhcp pool; fix the network statement in pool V20

W IPv6 ACLs

  1. Create an IPv6 ACL that permits SSH from 2001:db8:10::/64 to server 2001:db8:99::10 and denies all else. Attach inbound on the server facing interface. Provide the lines.
  2. Allow ICMPv6 for ND and PMTU but block other ICMPv6 types from untrusted VLANs. Provide example lines and the show that prints counters.
  3. Verify which interfaces have IPv6 ACLs applied and their directions. Name the show command.
Show answers

ipv6 access-list V6-SSH; permit tcp 2001:db8:10::/64 host 2001:db8:99::10 eq 22; deny ipv6 any host 2001:db8:99::10; interface g0/1 ipv6 traffic-filter V6-SSH in

permit icmp any any nd-na; permit icmp any any nd-ns; permit icmp any any packet-too-big; deny icmp any any; verify with show ipv6 access-list

show running-config | include ipv6 traffic-filter or show ipv6 access-list

X Troubleshooting Tickets

  1. VLAN 10 users cannot reach the default gateway. Trunk shows native VLAN mismatch. Provide the single fix line and one show to prove it.
  2. OSPF neighbors flap every few minutes on a broadcast network. DR changes frequently. Provide the fix to set both routers to priority 0 and the show to prove no DR election.
  3. NAT translations overflow and users lose connectivity. Provide the command to view stats and two changes that reduce misses without new IPs.
Show answers

switchport trunk native vlan 99 on both ends or match to 1; verify with show interfaces trunk

ip ospf priority 0 on both interfaces; verify with show ip ospf interface

show ip nat statistics; shorten timeouts and use interface overload instead of pool or enable DNS guard
